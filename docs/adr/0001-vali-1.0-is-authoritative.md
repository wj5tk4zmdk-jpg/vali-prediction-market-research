# ADR 0001: VALI 1.0 Is Authoritative

## Status

Accepted - 2026-06-23

## Context

The repository contains a useful research implementation but previously lacked
a local statement identifying the controlling methodology. Provider-specific
features or later experiments must not silently redefine the core.

## Decision

VALI 1.0 controls the core definitions of public Behavioral Attention `A`,
public/executable Priced Conviction `P`, `gA`, `gP`, signed divergence `S_t`,
divergence magnitude `M_t`, regimes, Clear Horizon, liquidity filters,
walk-forward validation, and execution-aware simulation. Methodology changes
require explicit approval, an ADR, and tests. `P_flow` is not part of the core.

## Consequences

Adapters must normalize into the core contract rather than alter it. Refactors
preserve existing behavior unless an approved, test-backed correction is being
made. Experimental extensions must remain clearly separated from VALI 1.0.
